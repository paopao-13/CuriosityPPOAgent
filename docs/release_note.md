# CuriosityPPOAgent 版本发布日志

> **项目名称**：CuriosityPPOAgent — ICM+RND 分层新颖信号融合好奇心驱动 PPO 智能体
> **开源协议**：MIT
> **技术栈**：PyTorch · gymnasium · crafter · minigrid · ale-py · wandb · ONNX · Vite+React
> **硬件基准**：AMD R7 6800H + RTX 3060 Laptop 6GB（显存峰值约 2.2GB）

本项目自 2024-06 启动，至 2026-07 完成 v1.0.0 百万步性能验收，累计 **12 次 Git 提交**，产出 63 个 Python 源码、7 个 YAML 配置、10 篇文档、22 份 Web Demo 文件与 144 项单元测试。

配置文件位于 `experiments/` 目录，训练入口 `scripts/train.py`，评测入口 `scripts/evaluate.py`。

## v0.3.0 — ICM 逆动力学模块

- **日期范围**：2024-08
- **Git 提交数**：1

**主要变更**：
- 实现 ICM（Intrinsic Curiosity Module）逆动力学模块
- 4 层 CNN 编码器将 (s, s') 映射至 288 维嵌入空间
- 前向预测损失：基于动作预测下一状态嵌入的 MSE 误差，衡量预测难度即内在好奇心
- 逆动力学损失：基于 (s, s') 预测动作的 Softmax 交叉熵，约束特征只保留可控信息
- 内在奖励 `r_int = 0.5 * MSE(pred_phi', true_phi')`，接入 PPO 奖励通道
- **里程碑**：Crafter 环境上 ICM 内在奖励信号正常生效

## v0.4.0 — RND 随机网络蒸馏模块

- **日期范围**：2024-09
- **Git 提交数**：1

**主要变更**：
- 实现 RND（Random Network Distillation）随机网络蒸馏模块
- `Target` 网络：固定随机初始化的 CNN，输出目标嵌入（不参与训练）
- `Predictor` 网络：可训练 CNN，拟合 Target 输出
- 内在奖励为两者输出的 MSE 误差，误差越大说明状态越新颖
- Target/Predictor 参数化与编码器共享开关，支持冻结 Target
- **里程碑**：RND 与 ICM 信号可独立/联合接入奖励通道

## v0.5.0 — Episodic Memory 情景记忆

- **日期范围**：2024-10
- **Git 提交数**：1

**主要变更**：
- 实现 Episodic Memory 情景记忆模块，基于 CPU-FAISS KNN 近邻检索
- 维护每个 episode 的状态嵌入库，采用 LRU 策略上限 200 条
- 通过 KNN 距离计算情景新颖度，episode 结束自动清空
- FAISS 索引构建在 CPU 侧，避免占用 GPU 显存
- **里程碑**：短时尺度（episode 内）新颖信号可用

## v0.6.0 — NGU 长短新颖信号融合

- **日期范围**：2024-12
- **Git 提交数**：1

**主要变更**：
- 实现 NGU（Never Give Up）风格长短时新颖信号融合
- 长时信号：RND 误差（跨 episode 持久）
- 短时信号：Episodic Memory 情景新颖度（episode 内衰减）
- 融合公式：`r_int = η × ICM + r_epi × min(max(α, 1), L)`
  - `η`：ICM 权重系数
  - `r_epi`：情景新颖度
  - `α`：可学习/可配置的比例因子
  - `L`：状态访问计数倒数，防止奖励爆炸
- 提供配置项控制各信号权重与裁剪范围
- **里程碑**：分层好奇心信号融合，探索效率显著提升
